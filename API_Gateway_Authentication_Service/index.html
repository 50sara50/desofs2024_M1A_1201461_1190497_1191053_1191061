Overview

How to use

A subscription management API providing four main abstractions: plans,users, and dashboard

Check the feature list for a description of the API's capabilities.

REST
Check the Swagger UI for the OpenAPI specification.

This is a REST/HTTP API that falls in Level 3 of the Richardson's Maturity Model adopting the REST constraints:

  • Client/server architecture
  • Statelessness
  • Cacheability
  • Layered
    • Resource identification
    • Resource representations - supporting multiple media types, e.g., application/json and application/vnd.pagsousa.hyper+json
    • Self-descriptive messages
    • HATEOAS - serving Link headers and links property

Besides serving application/json the application main media type is application/vnd.pagsousa.hyper+json retrieved from the project "A board for U" by Paulo Gandra . You can interactively explore the API and its documentation.

The following diagram depicts the Domain Application Protocol (DAP) of the service

The HTTP Link header is sent for each resource item (with rel, e.g., self, plan, collection (with, e.g., self, start, prev, next) or subresource like subscription/:subscriptionID/plan (with, e.g., plan). The presence of the link in the response representation depends on the current state of the resource, for instance, renew links in subscription are only sent for authenticated and authorized requests.

The following Custom Link Relations are used in the media type application/vnd.pagsousa.hyper+json

  • http://localhost:3000/rel/login - Obtain an access token
  • http://localhost:3000/rel/users - User collection
  • http://localhost:3000/rel/subscriptions - Subscriptions collection
  • http://localhost:3000/rel/plans - Plans collection
